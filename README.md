Heurísticas de Nielsen aplicadas
Sobre

Neste projeto, foram aplicadas três heurísticas de Nielsen para melhorar a interação com o usuário, mesmo sendo um sistema simples em console.

As heurísticas usadas foram:

Visibilidade do status do sistema
Prevenção de erros
Estética e design minimalista
Visibilidade do status do sistema

O programa mostra o que está acontecendo o tempo todo.

Exemplo:

Verificando CPU...
Lendo Memória RAM...
Finalizando...

Isso ajuda o usuário a entender que o sistema está funcionando e não travou.

Prevenção de erros

Com o uso de try-catch, o sistema evita que erros quebrem o programa.

Ao invés de travar, ele trata o erro e pode mostrar uma mensagem mais clara.

Isso melhora a experiência e evita frustração.

Estética e design minimalista

Mesmo sendo no console, o sistema tenta manter uma interface limpa:

Uso de cores (título em ciano, sucesso em verde)
Organização do texto
Sem excesso de informação na tela

Isso deixa mais fácil de ler e entender.

Conclusão

Mesmo sendo um projeto simples, aplicar essas heurísticas já melhora bastante a experiência do usuário.

Mostra que não é só fazer o sistema funcionar, mas também pensar em quem vai usar.
